1. Introduction

1.1 About Getaways

Getaways is a comprehensive online marketplace platform that connects travelers with hosts offering accommodations, experiences, and services. The platform facilitates bookings, payments, communications, and reviews in a seamless, user-friendly environment.

1.2 Platform Overview

  • Accommodations: Book unique places to stay
  • Experiences: Discover and book local experiences
  • Services: Find and book various services
  • GetPay Wallet: Integrated e-wallet system for payments
  • Messaging System: Direct communication between guests and hosts
  • Review System: Rate and review bookings

1.3 User Roles

  • Guest: Browse, search, book, and review listings
  • Host: Create and manage listings, handle bookings, earn income
  • Admin: Manage platform operations, users, bookings, and policies

2. Getting Started

2.1 Account Creation

For Guests:

  1. Click “Sign Up” on the homepage
  2. Choose “I’m a Guest” account type
  3. Enter your email address and password
  4. Provide your first name and last name
  5. Accept terms and conditions
  6. Click “Create Account”
  7. Verify your email address (check your inbox)
  8. Click the verification link in the email

For Hosts:

  1. Click “Sign Up” on the homepage
  2. Choose “I’m a Host” account type
  3. Complete the registration form
  4. Verify your email address
  5. After verification, you’ll be prompted to select your hosting type:
    • Accommodation
    • Experience
    • Service
  6. Begin the onboarding process

Alternative: Google Sign-In

  • Click “Continue with Google”
  • Select your Google account
  • Choose your account type (Guest or Host)
  • Complete profile setup

2.2 Email Verification

  • Why it’s required: Email verification ensures account security and enables full platform access
  • How to verify:
    1. Check your email inbox after registration
    2. Click the verification link in the email
    3. You’ll be redirected to the platform
    4. If you didn’t receive the email, check spam folder or request a new verification email

2.3 Password Reset

  1. Click “Forgot Password” on the login page
  2. Enter your registered email address
  3. Check your email for reset instructions
  4. Click the reset link (valid for 1 hour)
  5. Enter your new password
  6. Confirm your new password
  7. Log in with your new password

2.4 Login

  1. Navigate to the login page
  2. Enter your email and password
  3. Optionally check “Remember Me” for persistent login
  4. Click “Log In”
  5. You’ll be redirected based on your role:
    • Guests → Guest Dashboard
    • Hosts → Host Dashboard
    • Admins → Admin Dashboard

Step 1: Select Dates and Guests

  1. Choose check-in and check-out dates
  2. Select number of guests
  3. Review pricing breakdown

Step 2: Review Booking Details

  • Base price

  2. Click “Apply”

    1. Go to booking details
    2. Click “Complete Payment”

3.2 Security Rules

Firestore Rules

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read their own data, admins can read all
    match /users/{userId} {
      allow read: if request.auth != null && 
        (request.auth.uid == userId || 
         get(/databases/$(database)/documents/users/$(request.auth.uid)).data.roles.hasAny(['admin']));
      allow write: if request.auth != null && request.auth.uid == userId;
    }
    
    // Listings: public read, owner write
    match /listings/{listingId} {
      allow read: if true;
      allow write: if request.auth != null && 
        resource.data.ownerId == request.auth.uid;
    }
    
    // Bookings: participants and admins can read
    match /bookings/{bookingId} {
      allow read: if request.auth != null && 
        (resource.data.guestId == request.auth.uid || 
         resource.data.ownerId == request.auth.uid ||
         get(/databases/$(database)/documents/users/$(request.auth.uid)).data.roles.hasAny(['admin']));
      allow create: if request.auth != null;
      allow update: if request.auth != null && 
        (resource.data.guestId == request.auth.uid || 
         resource.data.ownerId == request.auth.uid ||
         get(/databases/$(database)/documents/users/$(request.auth.uid)).data.roles.hasAny(['admin']));
    }
    
    // Similar rules for other collections...
  }
}

Storage Rules

rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null && 
        request.resource.size < 5 * 1024 * 1024; // 5MB limit
    }
  }
